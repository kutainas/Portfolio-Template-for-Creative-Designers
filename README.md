# Portfolio Template para Diseñadores Creativos

Este proyecto consiste en una **plantilla de portfolio preparada para producción**, diseñada para mostrar trabajos creativos donde concepto, diseño y ejecución van de la mano.

---

Esta sección indica qué partes del proyecto pueden modificarse para personalizar el portfolio sin romper su estructura ni su diseño.

### Textos y contenido

- Archivos: `index.html`, `proyectos.html`  
- Se pueden modificar títulos, descripciones y textos de cada sección.  
- Los proyectos pueden duplicarse o eliminarse siguiendo la estructura existente.

### Imágenes, vídeos y documentos PDF

- Archivos: `index.html`, `proyectos.html`  
- Las URLs de los recursos multimedia pueden sustituirse directamente.  
- El sistema de modal ya está preparado para mostrar imágenes, vídeos y PDFs.

### Colores y branding

- Archivo: `css/base.css`  
- Las variables CSS definidas en `:root` permiten modificar colores principales, fondos y acentos visuales.

### Tipografía

- Archivo: `css/base.css`  
- Se pueden cambiar las fuentes y los tamaños base manteniendo la jerarquía visual del diseño.

### Animaciones y transiciones

- Archivo: `css/components.css`  
- Las duraciones y efectos de las animaciones pueden ajustarse sin alterar el layout general.

### Modal fullscreen

- Archivo: `js/modal.js`  
- Se puede personalizar:
  - el tamaño máximo del modal  
  - el comportamiento del cierre  
  - la reproducción de vídeos  
  - la navegación de documentos PDF  

---

## Accesibilidad

El proyecto incluye mejoras básicas de accesibilidad:

- Uso de HTML semántico  
- Textos alternativos en imágenes  
- Roles ARIA en el modal fullscreen  
- Navegación básica mediante teclado  

Estas mejoras pueden ampliarse según las necesidades del proyecto o los requisitos del cliente.

---

## Control de versiones y despliegue

El proyecto utiliza Git como sistema de control de versiones, siguiendo buenas prácticas:

- Uso de ramas para desarrollo  
- Commits descriptivos  
- Repositorio preparado para despliegue mediante GitHub Pages u otros servicios similares  

---

## Personalización y uso

Este template está pensado para ser reutilizado y adaptado fácilmente.  
Se recomienda el siguiente flujo de trabajo:

1. Clonar el repositorio  
2. Modificar textos, media y colores según las necesidades  
3. Revisar el apartado CUSTOMIZATION MAP  
4. Probar el diseño en distintos dispositivos  
5. Desplegar el proyecto en producción
